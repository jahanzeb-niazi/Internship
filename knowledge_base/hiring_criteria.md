# Hiring Criteria — General Guidelines

## Company Hiring Philosophy
We hire for potential and demonstrated ability. Degrees matter less than what you can build. We value:
- **Builders** who ship production code
- **Learners** who adapt to new technologies
- **Communicators** who collaborate effectively
- **Problem solvers** who think systematically

## General Qualification Criteria

### Must-Have (All Roles)
1. Minimum 2 years of professional software development experience
2. Proficiency in at least one primary language for the role
3. Experience with version control (Git)
4. Ability to write clean, testable code
5. Basic understanding of cloud services

### Strong Signals (Positive)
- Open-source contributions or public portfolio
- Experience with CI/CD and DevOps practices
- Knowledge of system design principles
- Self-directed learning (courses, certifications, side projects)
- AI/ML familiarity is increasingly valued across all roles

### Red Flags (Negative)
- No demonstrable work experience or projects
- Cannot name specific technologies or tools used
- Application is generic and not tailored to the role
- Claims skills without context or evidence
- Significant gaps with no explanation

## Role-Specific Minimum Requirements

### Backend Engineer
- Node.js or Python proficiency
- REST API development experience
- Database experience (SQL + NoSQL)
- Cloud deployment experience

### Frontend Engineer
- React.js or Vue.js proficiency
- Strong CSS/JavaScript skills
- Responsive design experience
- Testing framework experience

### Data Engineer
- Python and SQL proficiency
- ETL/pipeline experience
- Cloud data services experience
- Big data tools familiarity

## Interview Scheduling Criteria
- Qualified candidates should be scheduled within 5 business days
- Priority scheduling for candidates with:
  - 4+ years of experience
  - Matching preferred skills
  - Strong culture signals
- Borderline candidates require manager review before scheduling

## Diversity & Inclusion
- Evaluate candidates on skills and potential, not background
- Consider non-traditional paths (bootcamps, self-taught, career changers)
- Ensure fair scoring across all demographics
